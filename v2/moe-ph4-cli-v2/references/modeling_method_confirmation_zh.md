# 建模方法确认与结构选择规则

## 目标

在构建药效团前，先判断应该采用哪一种建模路线，以及需要几个 PDB 或几个构象参与建模。这个步骤决定最终模型是单结构模型、多结构 consensus 模型、配体训练集模型，还是混合验证模型。

## 一句话原则

先确定“证据类型”，再确定“结构数量”，最后确定“feature 选择目标”。不要因为手里有一个 PDB 就默认做单结构模型，也不要因为有多个 PDB 就机械叠加所有点。

## 方法路线判断

### 1. 单结构结构型模型

适用条件：

- 只有一个可信复合物结构。
- 配体、肽、辅因子或关键水/金属清楚可见。
- 结合位点刚性较强，缺少明显构象切换。
- 目标只是得到一个当前构象下的可筛选 query。

PDB 数量：

- 1 个主 PDB。
- 可另取 1 个 apo 或近似结构做构象风险检查，但不直接并入 consensus。

输出策略：

- 使用 `moe_native_site_ph4.js` 生成 raw candidates。
- 使用机制压缩得到 selected model。
- 报告中标记为“单构象结构证据模型”。

### 2. 多结构 consensus 模型

适用条件：

- 同一结合位点有 2 个以上高质量复合物结构。
- 配体/肽/片段存在构象差异或化学多样性。
- 公开证据中的模型明显来自多个复合物、多个 pose 或多个构象。
- 需要保留跨构象稳定热点，而不是某一个结构的偶然接触。

PDB 数量：

- 优先 3 个左右高质量代表结构。
- 少于 2 个不能称为 consensus。
- 多于 5 个时必须先聚类构象，避免冗余结构把某类热点权重放大。

结构选择优先级：

1. 同一靶点、同一位点、同一功能状态的复合物。
2. 分辨率高、缺失少、配体或肽完整。
3. 含关键辅因子、金属、水或修饰基团的结构优先保留。
4. 配体化学或肽序列有代表性差异。
5. 结合模式明确，避免明显错位、晶体接触主导或低可信 pose。

输出策略：

- 每个结构先独立生成 raw CSV 和 raw `.ph4`。
- 以受体共同 `CA` 或稳定核心原子叠合到参考结构。
- 转换 feature 坐标后聚类，保留跨结构稳定热点。
- 使用 `consensus_multistructure_ph4.js` 输出 consensus `.ph4`。

### 3. 配体训练集模型

适用条件：

- 缺少可靠复合物结构。
- 有一组活性/非活性或强弱活性配体。
- 目标是解释 SAR 或进行 ligand-based screening。

PDB 数量：

- 不强制需要 PDB。
- 若有结构，只作为合理性检查，不作为坐标真值。

输出策略：

- 使用配体叠合、活性约束、GH/enrichment 或 decoy 验证。
- 不把训练集模型与结构型模型混为一谈。

### 4. 混合验证模型

适用条件：

- 既有结构证据，又有活性/非活性集合、对接、MD 或 SAR 证据。
- 需要从多个候选药效团中挑选最能筛出真实活性分子的模型。

PDB 数量：

- 结构型部分按单结构或多结构规则选择。
- 验证集数量按活性/decoy 可用性决定。

输出策略：

- 药效团匹配只是第一层。
- 后续必须结合 docking pose、RMSD、GH score、enrichment、SAR 或 MD 稳定性。

## 特征数量判断

默认不要预设固定数量。数量由以下证据共同决定：

1. 独立机制区域数量：每个独立子口袋通常 1 个代表点。
2. 强化学基团数量：强阴离子、强阳离子、芳香面等可多点表达，但必须代表不同空间方向。
3. 多结构稳定性：多个结构共同出现的点优先。
4. 筛选必要性：删除后筛选假设不变的点应压缩。
5. 验证目标：若需要复现既有目标模型的 feature-class 组成，可把该组成作为 target quota，但仍需保留机制解释。

常见范围：

- 小分子刚性口袋：4-7 个。
- 肽/蛋白-蛋白界面：6-10 个。
- 多结构 consensus 或修饰肽模型：可到 8-12 个，但必须证明每个点属于独立机制或稳定热点。

## 从公开研究资料中提取经验

可以提取的是通用经验：

- 采用了单结构还是多结构。
- 是否保留 mixed feature，例如 `Ani&Acc`、`Don&Acc`。
- 是否显式使用水、金属、辅因子或修饰基团。
- feature 数量与验证指标之间的关系。
- 是否使用 RMSD、GH score、enrichment、docking 或 MD 作为二次筛选。

不能机械提取的是：

- 特定靶点名、PDB ID、残基号、feature 编号。
- 只因某个案例出现过就固定保留某个 family。
- 未经结构/验证支持的人工选择。

## 建模前检查清单

- 是否清楚模型类型：单结构、多结构、配体训练集、混合验证。
- 是否说明 PDB 选择理由。
- 是否说明排除哪些 PDB，以及排除理由。
- 是否说明是否处理水、金属、辅因子、修饰残基。
- 是否说明 feature perspective：ligand-side、receptor-complement 或 mixed。
- 是否说明最终 feature 数量的依据。
- 是否有验证计划：strict/region、RMSD、GH/enrichment、docking 或 SAR。
