# v2 机制压缩深层规则

## 总原则

药效团不是把 MOE 或接触图里所有点照抄出来，而是把结合机制压缩成最少、最独立、最能区分活性分子的空间化学约束。2.0 的改进重点是：原始候选由 MOE 提供，最终 selected 由机制压缩决定。

## 1. 先锁定 feature perspective

多数 Query Pharmacophore 用于筛分新分子，因此 feature 是候选分子应该呈现的功能团。判断时优先问：

- 这个点是在描述配体/肽侧的供受体、芳香、疏水、正负电，还是受体互补环境？
- 公开资料是否从共晶配体、底物肽、抑制剂 pose 中提取 query？
- 如果是 ligand-side query，受体上的芳香墙可投影成配体侧 `Aro/Hyd`，但不要把受体芳香残基本身当成候选分子的芳香点。

视角不锁定会导致最常见退步：MOE 看到了接触，但 selected 选择了互补方向错误的 feature。

## 2. 强化学基团优先保留其主导家族

- 磷酸、硫酸、羧酸、核苷酸样强阴离子：优先按 `Neg`，可兼容 `HBA`，但不能只当普通 HBA。
- 胍基、季铵、质子化胺：优先按 `Pos`，再看是否同时具有 HBD 方向性。
- 酰胺、脲、骨架羰基：优先区分 HBA/HBD 方向；不要因邻近疏水面而改成 Hyd。
- 芳香环若承担口袋形状和 π 面识别，优先保留 `Aro`；若只是非方向性埋藏，可压缩成 `Hyd`。

## 3. 子口袋分区比残基名字更重要

不要记具体蛋白名、PDB、残基号。用可迁移的区域语言：

- `buried_nonpolar_anchor`：深埋非极性锚点，通常对命中选择性贡献大。
- `polar_gate`：口袋入口或转角处的方向性氢键/电荷门。
- `anionic_cap` 或 `cationic_cap`：强电荷头部的锚定区。
- `aromatic_wall_projection`：受体芳香/π 面提供的形状约束，在配体侧投影为 Aro/Hyd。
- `backbone_register`：肽或底物样分子的主链走向约束，常由少数 HBA/HBD 点表达。
- `solvent_exposed_tolerant`：溶剂暴露、可替代、对筛选贡献弱的区域。

同一区域只保留最能代表机制的 1 个点；相邻但属于不同物理机制的区域不能强行合并。

## 4. 倾斜口袋与投影规则

遇到配体/肽侧链倾斜进入口袋时，药效团点不应机械放在最近受体原子上，而应放在候选分子功能团的可替代中心：

- 倾斜指向疏水深处：保留 `Hyd`，中心放在侧链/疏水团可替代体积的质心附近。
- 倾斜贴靠芳香/π 墙：若形状和方向性约束明显，保留 `Aro`；否则压缩为 `Hyd`。
- 倾斜末端形成盐桥或强氢键：保留末端 `Pos/Neg/HBD/HBA`，疏水柄只在其本身提供筛选能力时保留。
- 若一个倾斜侧链同时有疏水柄和电荷头，必须判断两者是否属于两个独立子口袋；不是独立子口袋时不要同时过度保留。

## 5. 证据权重

优先级从高到低：

1. 共晶/稳定 pose 中的埋藏、方向性、低可替代热点。
2. 多构象、多类似物或 MD 稳定出现的热点。
3. 与已知 SAR 一致的必要官能团。
4. MOE 原始注释中重复出现但缺少机制解释的点。
5. 暴露、溶剂可替代、只描述大小而不描述相互作用的点。

## 6. selected 压缩规则

- selected 不是越多越好。公开资料模型常把候选池压缩成 4-7 个高信息点。
- 同一功能家族在同一子口袋重复出现时，保留更埋藏、更方向性、更靠近功能团中心的点。
- 如果删除某点不改变可筛选的化学假设，它应降级或删除。
- 如果某点虽然分数不高，但代表唯一子口袋或唯一电荷/芳香机制，应被机制规则提升。

## 7. 失败诊断

- `full candidates region` 高而 `selected strict` 低：MOE/结构层已经看见了，问题在压缩排序和 feature perspective。
- `strict` 低但 `region` 高：掌握了区域规律，但坐标中心、家族别名或公开资料视角还需校准。
- `region` 低：输入结构、位点、链、构象、水/金属/辅因子或 MOE 原始注释存在上游问题。
- selected 数量明显多于公开资料：通常是冗余未合并；明显少于公开资料：通常漏掉了独立子口袋或把强家族过度压缩。
